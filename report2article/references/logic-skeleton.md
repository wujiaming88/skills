# 逻辑骨架模板

报告是"平行罗列"（按对象/维度/分组），文章需要一条**贯穿的逻辑主线**把信息串起来。
下面是可选骨架——**选一种作为主线，不要堆叠**。所有骨架都遵守红线：只重组报告已有信息，不新增判断。

## 通用四段骨架（默认）

1. **为什么值得读**（开篇导读，1-3 句）
   - 提炼本期/本文最值得关注的几个看点，给读者读下去的理由。
   - 🔴 只能用报告里**已有的结论**提炼，不得新增判断或拔高。
2. **是什么 / 发生了什么**（主体）
   - 把报告核心事实按逻辑分组呈现（不是按报告原顺序照搬）。
3. **怎么演变 / 彼此什么关系**（串联）
   - 用过渡句把孤立的点连成线，揭示报告中**已隐含**的关联/对比/趋势。
4. **意味着什么**（收束）
   - 收拢报告里**已有的**影响判断、趋势研判。不加新观点。

## 可选主线（按内容性质择一）

| 主线 | 适用 | 组织方式 |
|---|---|---|
| 时间线 | 一个主题的演变历程 | 按时间顺序串联各节点事实 |
| 冲突/对比 | 多家厂商/多种方案竞争 | 按"争夺什么→各家怎么打→格局如何"组织 |
| 主题递进 | 一个领域多个子问题 | 从基础概念递进到深水区 |
| 总分总 | 信息量大、对象多 | 先给全局速查/导航，再分块详述，最后收束 |

## 重组手法

- **打散四段式**：报告的「对象A：动态/数据/链接/判断 → 对象B：…」结构，重排为按主题/冲突/时间线的叙事，把每个对象的信息**分散**到对应主题下——但每条信息都要落到文章某处，**不丢失**。
- **融合而非罗列**：报告的「本周动态 / 关键数据 / 原文链接 / 影响判断」四段，融合为连贯段落（数据自然嵌入正文，链接以行内或脚注保留），四类信息一个都不能少。
- **承上启下句**：段落之间用纯导航过渡句黏合，让读者顺着一条线走，而不是跳读表格。过渡句不得携带新信息。

## 注意

- 速查表/导航表只是"索引"，不替代下方深度正文。表里一句话 ≠ 正文只写一句话。
- 周报场景下，深度内容（每个有动态对象的 ≥200 字笔记）必须全量进文章，编排只改组织和标题，不砍深度。
