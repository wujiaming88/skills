# 信息零丢失校验清单模板

转换完成后，用这份清单**逐条比对**原报告与文章。这是红线的硬执行手段。

## Step 1：从报告提取信息清单（转换前先做）

通读报告，逐项列出：

```
## 对象清单
- [ ] 对象1：________
- [ ] 对象2：________
...

## 数据点清单（数字/日期/名称/估值/版本号等）
- [ ] ________（出处：______）
- [ ] ________（出处：______）
...

## 结论/判断清单（注意记录强弱程度）
- [ ] ________（程度：可能/已经/必然…）
...

## 链接/出处清单
- [ ] ________
...
```

## Step 2：比对文章（转换后逐条核对）

| 维度 | 检查 | 通过标准 | 核对方式 |
|---|---|---|---|
| ① 对象覆盖 | 报告每个对象，文章有对应位置？ | 全部有 | 🔧 可 grep 对象名 |
| ② 事实/数据覆盖 | 每个数据点都在且数值一致？ | 零偏差 | 🔧 可 grep 数字/日期 |
| ③ 结论/判断覆盖 | 每条判断都在，且**强弱未变**？ | 程度未被改强/改弱 | 👁 **只能逐行人工比对**（"可能→已经"这类漂移 grep 不出来） |
| ④ 链接覆盖 | 每个原文链接/出处都保留？ | 全部在（重复 URL 按去重计数；零链接则 L=0） | 🔧 可 grep URL |
| ⑤ 零新增 | 文章有无报告里找不到出处的事实/观点/数据？ | 一个都没有 | 👁 **只能逐行人工比对**（过渡句夹带的新判断 grep 不出来） |

> 🔴 别被"机械核对"给假信心：③和⑤是**语义判断**，字符串搜索搜不出来，必须把文章句子和报告原句一句一句对读。①②④才是 grep 能兜住的。

## Step 2.5：独立反向交叉复核（必过闸，不能省）

`文章覆盖 == 清单` 只证明"文章覆盖了我提取的清单"，**不证明"清单覆盖了整份报告"**。同一个人既提清单又写文章，提取时漏掉的对象会让清单天生就短，Step 2 照样"全部对应 ✅"——信息其实丢了。所以必须加一道**独立**反查：

- **对抗式反查**：抱着"信息一定丢了"的假设，**重新打开原始报告**（不看清单、不凭记忆），逐段扫描，专找"清单/文章里没有的对象、数字、链接、判断"。找到即补进清单与文章。
- **长报告二次派生清单**：末尾**从原始报告重新独立派生一份对象清单**，与 Step 1 那份逐条比对，**两份必须一致**；不一致即以原始报告为准补齐（细则见 `long-report-protocol.md`）。

## Step 3：交付时报出校验结果

链接计数说明：**重复出现的同一 URL 按去重后的唯一数计入 L**；报告**零链接**时 L=0，写"无原文链接（报告本身未附）"，不得凭空补链接。

固定格式（必须出现在交付说明里）：

```
信息校验：N 个对象 / M 个数据点 / K 条结论 / L 个链接，全部对应 ✅
```

## 不过闸处理

- ②③④任一条发现**遗漏** → 把缺失信息补回文章对应位置。
- ⑤发现**新增** → 删除该越线内容（或核对是否报告里其实有，只是被改写）。
- ③发现**强弱被改**（如"可能"被写成"已经"）→ 改回报告原义。
- 任一条不过 → 打回修正，**不得发布**。
