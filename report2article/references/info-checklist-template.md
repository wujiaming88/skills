# 信息零丢失校验清单模板

转换完成后，用这份清单**逐条比对**原报告与文章。这是红线的硬执行手段。

## Step 1：从报告提取信息清单（转换前先做）

通读报告，逐项列出：

```
## 对象清单
- [ ] 对象1：________
- [ ] 对象2：________
...

## 数据点清单（数字/日期/名称/估值/版本号等）
- [ ] ________（出处：______）
- [ ] ________（出处：______）
...

## 结论/判断清单（注意记录强弱程度）
- [ ] ________（程度：可能/已经/必然…）
...

## 链接/出处清单
- [ ] ________
...
```

## Step 2：比对文章（转换后逐条核对）

| 维度 | 检查 | 通过标准 |
|---|---|---|
| ① 对象覆盖 | 报告每个对象，文章有对应位置？ | 全部有 |
| ② 事实/数据覆盖 | 每个数据点都在且数值一致？ | 零偏差 |
| ③ 结论/判断覆盖 | 每条判断都在，且**强弱未变**？ | 程度未被改强/改弱 |
| ④ 链接覆盖 | 每个原文链接/出处都保留？ | 全部在 |
| ⑤ 零新增 | 文章有无报告里找不到出处的事实/观点/数据？ | 一个都没有 |

## Step 3：交付时报出校验结果

固定格式（必须出现在交付说明里）：

```
信息校验：N 个对象 / M 个数据点 / K 条结论 / L 个链接，全部对应 ✅
```

## 不过闸处理

- ②③④任一条发现**遗漏** → 把缺失信息补回文章对应位置。
- ⑤发现**新增** → 删除该越线内容（或核对是否报告里其实有，只是被改写）。
- ③发现**强弱被改**（如"可能"被写成"已经"）→ 改回报告原义。
- 任一条不过 → 打回修正，**不得发布**。
