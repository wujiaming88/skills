# 超长报告强制处理协议

**强制要求**：报告再长，信息也不能丢。这是红线的延伸。

## 为什么需要它（两个失败模式）

| 失败模式 | 现象 | 后果 |
|---|---|---|
| **输入截断** | 报告太长，一次读不全，模型只看到前半段就开始编排 | 后半段对象/数据静默丢失 |
| **输出截断** | 编排后长文超单次输出上限（stopReason=length） | 文章写一半就断，尾部对象消失 |

光靠"注意别丢"挡不住。必须用**分块 + 销账**的机械流程。

## 何时触发本协议

满足任一条即按本协议处理（宁可多走流程，不可冒丢失风险）：
- 报告对象数 ≥ 8 个，或
- 报告正文预估 ≥ 6000 字 / ≥ 8000 token，或
- 一次读取被工具截断（read 提示 more lines / API 上限）。

## 五步强制流程

### Step 1：先建全量对象清单（处理前的"分母"）
- 分段读全报告（read 用 offset/limit 翻完，**绝不只读前半段**）。
- 列出全部对象/数据点编号清单：`#1 …  #2 …  … #N`。
- 这份清单是后面每一块的"销账依据"。记下总数 N。

### Step 2：分块切分
- 按报告的逻辑章节/分组把对象切成若干块（每块 2–4 个对象为宜）。
- 给每块标注它覆盖清单里的哪些编号（如"块二 = #4~#7"）。

### Step 3：分块编排 + 立即落盘（抗输出截断）
- 一次只编排一块，**编排完立即 write/append 写到输出文件**。
- 长文用**多次 append**累加，绝不囤在上下文里等最后一次性输出整篇。
- 每块落盘后确认文件确实增长（wc -l / tail 看尾部）。

### Step 4：逐块销账
- 每落盘一块，更新覆盖计数："已覆盖 X/N 对象（#编号清单）"。
- 处理到下一块前，确认上一块的所有编号都已写入文件。
- 缺哪个编号 → 回去补哪个，补完才推进。

### Step 5：末尾一致性复核 + 独立二次派生（必过闸）
- 全文落盘后，对照 Step 1 清单跑完整的信息零丢失 5 维校验。
- **必须 X == N**（清单总数 = 文章覆盖数）。
- 🔴 **独立二次派生清单**：**重新打开原始报告**（不复用 Step 1 那份、不凭记忆），从头再派生一份对象清单 N'。**要求 N' == N 且逐条一致**；若 N' > N，说明 Step 1 提取时就漏了对象——以原始报告为准补齐清单和文章，再重跑校验。这一步专门堵"同一次心智既建清单又写文章、漏提取时 N 天生就短"的漏洞。
- 用 grep 做**内容级**机械核对（对象名/关键数据/URL 逐个命中，见小抄）。
- 报出：`信息校验：N 对象 / M 数据点 / K 结论 / L 链接，全部对应 ✅`。

## 机械核对小抄

> ⚠️ 分清两类 grep，别搞反：
> - **A. 从原始报告提取基线**（对着**报告**跑）——数报告里有多少对象/链接，得出基线数量。
> - **B. 验证文章内容覆盖**（对着**文章**跑）——正确文章已把四段式（本周动态/关键数据/原文链接/影响判断）**打散融合成叙事**，**不该**再有这些结构标签，也大多没有 `### ` 对象头。所以文章侧**不能**grep 结构标签，只能 grep **内容特征**：对象名、具体数字/日期、URL。拿结构标签去验文章会误报"信息丢了"，还会逼你退回四段式、毁掉逻辑质量。

```bash
# —— A. 基线：对【原始报告】跑，得出该有多少 ——
grep -c '^### ' 原始报告          # 报告里的对象数（基线 N，视报告格式而定）
grep -o 'https://[^ ；）]*' 原始报告 | sort -u | wc -l   # 报告去重后唯一链接数（基线 L）

# —— B. 覆盖验证：对【文章】跑，逐个内容特征命中 ——
# 对象名（把清单里每个对象名代入，应命中 ≥1 次）
for name in 对象名1 对象名2 …; do echo "$name: $(grep -c "$name" 文章)"; done
# 关键数字/日期（把清单里每个数据点代入）
for d in "2美元" "2026-06-11" …; do echo "$d: $(grep -c "$d" 文章)"; done
# 文章去重后唯一链接数，应 == 基线 L
grep -o 'https://[^ ；）]*' 文章 | sort -u | wc -l
```

## 红线提醒

- 分块是**抗截断手段**，不是"写短/精简"的借口。每个对象该多深写多深，报告四段式的**信息**逐条保留（表达可按 `prose-style.md` 收紧融合，但不丢事实）。
- 分块只改"分几次写"，不改"写什么"——编排层动作不变，内容层一字不动。
