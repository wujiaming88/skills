# 软考系统架构设计师真题搜索

## 使用方式

用户提供以下参数（均可选，至少一个）：
- **知识领域**：如"设计模式"、"架构风格"、"数据库"、"信息安全"等
- **时间范围**：如"2024"、"2020-2025"、"最近三年"
- **科目类型**：综合知识 / 案例分析 / 论文（默认全部）

示例：
```
搜索真题：设计模式 2020-2025
搜索真题：微服务架构 案例分析
搜索真题：2024下半年 论文
搜索真题：信息安全 最近三年 综合知识
```

## 搜索流程

### 第1步：加载数据源配置
读取 `references/sources.md` 获取全部数据源和知识领域映射。

### 第2步：确定搜索范围
根据用户输入确定：
- 目标年份列表
- 目标知识领域关键词
- 目标科目类型

### 第3步：从仓库拉取（优先）

**GitHub 仓库**（优先使用，结构最清晰）：
```bash
# 列出指定年份真题文件
gh api "repos/xiaomabenten/system_architect/contents/03、历年真题(2009年-2025年)+答案解析/{年份}" --jq '.[].name'

# 获取文件内容（markdown/txt）
gh api "repos/xiaomabenten/system_architect/contents/{完整路径}" --jq '.download_url'
# 然后 web_fetch 下载内容
```

**Gitee 仓库**（备用/补充）：
```bash
# 列出目录
curl -s "https://gitee.com/api/v5/repos/zaonai/system_architect/contents/03、历年真题(2009年-2025年)+答案解析/{年份}"

# 获取文件下载链接
# 解析 JSON 中的 download_url 字段
```

**优先级**：xiaomabenten(GitHub) → zaonai(Gitee) → xxlllq(GitHub) → 备用Gitee源

### 第4步：Web搜索补充（按需）

当仓库中找不到目标内容，或用户需要特定知识点真题时：
```
web_search("系统架构设计师 {年份} {科目} 真题 {知识点}")
```
然后 `web_fetch` 抓取搜索结果页面内容。

### 第5步：整理输出

按以下格式输出搜索结果：

```
## 搜索结果：{知识领域} | {时间范围} | {科目}

### {年份}{上/下半年} - {科目}

**题目 X**：
（题目内容）

**选项**：（如果是选择题）
A. ...  B. ...  C. ...  D. ...

**答案**：X

**解析**：
（解析内容）

---
来源：{仓库名/URL}
```

## 注意事项

- PDF文件无法直接读取内容，标注文件路径和下载链接供用户下载
- 图片格式的真题标注为"图片格式"并提供链接
- 回忆版真题标注"回忆版，可能与原题有出入"
- 搜索结果按年份倒序排列（最新优先）
- 每次搜索最多展示10道相关题目，避免信息过载
