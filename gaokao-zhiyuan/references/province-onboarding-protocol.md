# 省份快速学习协议 — 面对任意省份，几步把规则学全

> 2026-06-28 老板要求：本 skill 面向**全国所有省份**，必须具备**快速学习并适配任意省份**志愿规则的通用能力，不能只硬编码某一省。
> 核心理念：**规则不靠记忆、靠现查现学。** 中国高考志愿规则各省差异大且逐年微调——唯一可靠做法是用一套**固定的「省份 onboarding 清单」**，针对考生所在省**当年**把规则查全、结构化，再套通用推演引擎出表。
> 这是 Step 0.5 强制前置步骤：**任何省份开工前，先跑完本协议把该省「规则卡」建好，再进数据采集与推演。**

---

## 为什么必须现查现学（不能背规则）

1. **省际差异大**：投档模式（院校专业组 / 专业+院校 / 传统院校）、志愿数量（30~112 不等）、是否有组内调剂、批次合并情况、选科要求，各省完全不同。
2. **逐年微调**：同一省份志愿数量、批次设置、代号编码、合并本科批次等**每年可能变**。去年的湖北 ≠ 今年的湖北。
3. **错规则 = 错志愿**：把 A 省规则套到 B 省（如把湖北"组内6专业+服从调剂"套到浙江"专业+院校无调剂"），会直接出一张废表。
4. 所以：**记住"怎么学"，而不是"学到的某省结论"。** 本协议就是"怎么学"。

## 🧭 省份规则卡 — 必须查全的 12 项（onboarding 清单）

针对考生「省份 + 当年年份」，逐项查实并填这张卡。**每项标来源**；查不到标「未查到（待官方）」，绝不臆造。

| # | 字段 | 说明 / 为什么要 | 主查处 |
|---|------|----------------|--------|
| 1 | **高考模式** | 3+1+2 / 3+3 / 传统文理 → 决定科类口径与选科 | 省考试院当年招生规定 |
| 2 | **投档模式** | A院校专业组 / B专业(类)+院校 / C传统院校志愿 → 决定录入单位与调剂逻辑 | 省考试院志愿设置文件 |
| 3 | **本科批次结构** | 是否合并本科批（一本二本合并）/ 有无特殊类型线 / 提前批构成 | 省考试院批次说明 |
| 4 | **本科批可填志愿数** | 30/45/80/96/112… → 决定冲稳保铺开密度 | 当年志愿设置文件 |
| 5 | **每个志愿单位内结构** | 模式A：每组≤N专业+是否服从调剂；模式B：1专业+1院校无调剂 | 同上 |
| 6 | **是否服从专业调剂** | 模式A有、模式B无 → 决定是否要逐组勾服从、退档风险话术 | 同上 |
| 7 | **代号编码规则** | 院校代号位数 + 专业组/专业代号构成（如湖北6位=4院校+2组号） | 当年招生计划册 |
| 8 | **官方填报系统网址** | 考生实际录入的系统 + 官方辅助查询系统 | 省考试院公告 |
| 9 | **填报/征集时间表** | 各批次起止、征集志愿时间 → 提醒考生别错过 | 省考试院时间安排 |
| 10 | **一分一段表口径** | 该省按什么分类发布（普通类首选物理/历史；旧高考文/理）→ 定位锚点只取普通类专表 | 省考试院一分一段表 |
| 11 | **当年批次线/特殊线** | 本科线、特殊招生线（独立核实，不采信口述/任务书） | 省考试院分数线公告 |
| 12 | **投档规则细则** | 平行志愿"分数优先遵循志愿一轮投档" / 顺序志愿权重 / 同分排序规则 | 省考试院投档规则 |

> 填完这 12 项 = 该省「规则卡」建成。后续 Step 2~5 全部基于这张卡执行。

## 🔎 快速学习的固定检索动作（怎么查最快查最准）

按官方源优先级（红线 5）逐步取，每步标来源：

1. **一把抓官方总文件**（最高效）：
   `web_search "<省份> <年份> 普通高校招生 网上填报志愿 考生须知/政策问答"`
   → 认准 **省教育考试院/招生考试院/教育厅** 官方域名；优先教育部阳光高考平台转载的官方问答（gaokao.chsi.com.cn）。这类「阳光招生政策问答」通常一篇就覆盖第 1~9、12 项。
2. **志愿设置专项**：`web_search "<省份> <年份> 志愿设置 院校专业组 可填几个 服从调剂"` → 补第 4/5/6 项精确数字。
3. **代号规则**：`web_search "<省份> <年份> 院校专业组代号 几位 招生计划册"` → 补第 7 项。
4. **系统网址**：官方问答里通常直接写明填报系统 URL（第 8 项）。
5. **一分一段表**：`web_search "<省份> <年份> 一分一段表 普通类 首选物理/历史"` → 第 10 项 + 考生位次定位。
6. **分数线**：`web_search "<省份> <年份> 本科批 录取控制分数线"` → 第 11 项。
7. 取数降级链（强制）：`web_fetch` 直取官方页 → 超时/被拦切 web-search-plus → 内置 web_search → x_search；多法都拿不到才标「未查到」。

> 效率提示：**第 1 步的「阳光招生政策问答」往往一篇覆盖大半张规则卡**，先抓它，再用 2~6 补缺口，通常 5~8 次检索就能建好一张完整规则卡。

## 🗂 通用模式适配（建好规则卡后，套对应玩法）

依规则卡第 2 项「投档模式」，套 `province-rules.md` + `system-entry-format.md` 对应分支：

- **模式 A 院校专业组**（湖北/湖南/广东/江苏/福建/河北/北京/重庆/辽宁等）
  → 录入单位=院校专业组；**必须查到专业组级位次**（不能只用校线最低）；逐组勾服从调剂；草表含「序号+组代号+组内≤N专业排序+服从」。
- **模式 B 专业(类)+院校**（浙江/山东等）
  → 录入单位=「专业+院校」组合，所见即所得、**无组内调剂**；冲稳保梯度要更密集（无调剂兜底）；草表含「序号+专业代号+院校代号」。
- **模式 C 传统文理+院校**（旧高考存量省）
  → 院校志愿+校内专业+服从调剂；顺序志愿省份第一志愿权重极高，冲档更保守。

> ⚠️ 数量决定冲稳保分配：志愿越多铺得越开，越少越保守。冲稳保**比例**用通用保守口径（稳+保占80%），但**绝对数量**按该省志愿数折算（45组→冲~9/稳~18/保~18；96个→按比例放大）。

## ✅ 省份适配自检（进 Step 2 前必过）

- [ ] 12 项规则卡逐项有值或明确标「未查到」，每项带来源
- [ ] 投档模式已确定，并选定对应 A/B/C 玩法与草表结构
- [ ] 本科批可填志愿数 = 当年官方数字（非套用别省/去年）
- [ ] 代号编码规则已查实（或标「以当年计划册为准」），未编造代号格式
- [ ] 官方填报系统网址已确认（考生真正录入的那个）
- [ ] 一分一段表口径=该省普通类对应科类专表
- [ ] 当年批次线已独立核实
- [ ] 所有数字均为**考生当年**该省官方，无跨省/跨年误用

> 一句话：**给任何省份出表，先建这张规则卡。规则卡不全 → 不许进推演。** 通用性 = 一套固定学习协议 × 一套固定推演引擎。
