# gaokao-zhiyuan — 中国高考志愿填报建议器

> 一套可复用的志愿填报方法论。定位：**面向全国所有省份的通用引擎**，基于真实官方数据 + 位次推演，给出冲/稳/保三档志愿建议。
> **通用性原则（老板 2026-06-28）**：不硬编码某一省。面对任意省份，靠**一套「省份快速学习协议」现查现学该省当年规则**（见 `references/province-onboarding-protocol.md`），再套**同一套位次推演引擎**出表。记住"怎么学"，不背"某省结论"。
> 本 skill 只承载「填报能力」；调用方的身份/记忆由各自 workspace 提供。

## 这个 skill 是什么

输入：考生**分数 + 省份 + 科类（物理/历史/3+3选科）+ 全省位次 + 填报偏好**（城市/专业/院校层次/排除项）。
输出：一份**含冲/稳/保三档、每校带近三年真实录取数据、带录取概率定位、带服从调剂提醒**的志愿填报建议表。

**定位：数据推演器，不是算命先生。** 每一所推荐院校的档位（冲/稳/保）都必须由**位次数学推演**得出，且建立在**官方数据 + 多源交叉验证**之上。

## 🔴 红线（最高优先级，违反即作废）

1. **禁止不经数据验证就给建议。** 任何一所院校进入建议表，必须先有该校在该省该科类**历年（强制 ≥2 年，力争 3 年）**真实录取位次——**单年数据不足以定档**，必须结合历年看趋势/大小年。
2. **禁止杜撰、臆造、估算冒充真实数据。** 拿不到的数据 → 明确标注「未查到」，绝不编造一个分数/位次填进去。
3. **所有建议必须三要素齐全：考生分数 + 全省位次 + 该院校在本省历年（≥2年）真实录取数据。** 缺位次锚点不得下档位结论。
4. **数据采信前必须严格核验三个维度（时间 / 省份 / 考生类别），任一不符即作废该数据**：
   - ⏱ **时间**：是否目标年份的录取数据？（聚合站常把 2024/2025/2026 混标；尚未发生的年份不存在录取数据）
   - 📍 **省份**：必须是**该院校在考生所在省的招生计划/录取数据**。考生在A省、报考B省的院校，只能用**该校在A省**的录取线/位次/计划，绝不能用该校在B省（本校所在省）或别省的数据冒充——同校跨省录取线差异巨大，且**该校若本年不在A省投放某专业/专业组，考生根本报不了**。报考前必核该校当年在A省有无相应招生计划。
   - 🎓 **考生类别**：是否考生对应科类/批次？（物理/历史、文/理、本科批/专项/中外合作组必须对得上）
   - 📊 **一分一段表口径（强制·定位锚点）**：定位考生位次**只能用「普通类」对应科类那张表（普通类-首选物理 / 普通类-首选历史）**。一分一段表按 **普通类 / 艺术类 / 体育类 / 技能高考** 分开发布，**绝不能用「总分一分一段统计表（含艺术/体育/技能混排）」的位次**——混排表把所有类别考生合在一起排名，同一分数对应的位次会显著偏前，直接导致定档系统性误判。取数时先确认表头写明「普通类（首选物理/历史）」，否则废弃重取。
5. **官方源优先级铁律**：省教育考试院/招生考试院/招生信息网官方投档线 + 院校官网招生网 > 权威聚合（中国教育在线/掌上高考等）> 其他。一手官方查得到的，不许用二手代替。
6. **冲/稳/保档位必须由位次推演得出**（见 `references/methodology-rank-anchor.md`），不许凭感觉/凭校名贴标签。
7. **录取概率必须综合三因子计算，禁杜撞臆想。** 估算录取概率/定档不准只看位次差，必须综合：①位次差 ②该校当年在本省招生名额增减 ③本省考生数/该科类上线人数变化；每个因子要有真实数据，缺则标“未查到”并以位次差为主取保守值，绝不用猜测值抬高概率（见 methodology「录取概率综合计算法」）。

> ⚠️ 实战血泪（务必内化，详见 `references/pitfalls-checklist.md`）：曾把一所「考生位次比该校最低录取线还低 1.2 万名」的院校误当"冲档"塞进表里——那不是"冲一冲能上"，是"超出能力范围根本够不到"。**冲档 = 考生位次略高于（差距可控）该校录取位次；不是无限拔高。**

## 怎么用（任何填报先走这一条主线）

### Step 0 — 锁定考生画像（缺一不可）
必须先确认 5 项，缺则向用户追问，**不得用默认值蒙混**：
1. **省份**（决定志愿规则 + 数据源）
2. **科类**：物理类/历史类（新高考3+1+2）或 文/理（旧高考）或 3+3 选科组合
3. **分数**
4. **全省位次（一分一段表定位）** — 这是跨年比较的唯一锚点，**比分数更重要**
5. **填报偏好**：意向城市、意向专业、院校层次底线、明确排除项（如"不要体育/师范"）

> 位次 > 分数：每年试题难度不同导致分数线漂移，**只有位次可跨年比较**。所有推演以位次为锚。

### Step 0.5 — 省份适配（强制前置，老板 2026-06-28）
**任何省份开工前，先跑完 `references/province-onboarding-protocol.md` 把该省当年「规则卡」建好。** 这是本 skill 通用性的根基：
- 按协议的 **12 项规则卡**逐项查实（高考模式/投档模式/本科批结构/可填志愿数/组内结构/是否服从调剂/代号编码/填报系统网址/时间表/一分一段口径/批次线/投档细则），每项标来源
- 用协议的**固定检索动作**快速学（先抓该省「阳光招生政策问答」一篇覆盖大半，再补缺口，通常 5~8 次检索建好）
- 依「投档模式」选定 A院校专业组 / B专业+院校 / C传统 对应玩法与草表结构
- **规则卡不全 → 不许进后续数据采集与推演**

### Step 1 — 读方法论内核（必读）
- `references/province-onboarding-protocol.md` — **省份快速学习协议**（通用性根基）：面对任意省份，用固定 12 项规则卡 + 固定检索动作现查现学该省当年规则，再套通用推演引擎
- `references/methodology-rank-anchor.md` — **位次锚定推演法**：冲/稳/保的数学定义（**默认保守口径**：冲靠前3%~8%/稳−3%~+5%/保靠后8%~25%，稳+保占有80%）、位次差换算、概率定位、专业组内部分差处理、梯度排布
- `references/system-entry-format.md` — **志愿系统录入格式适配**：让考生能把表原样录入官方系统（湖北样板：资讯平台 zspt.hubzs.com.cn、本科普通批 45 个院校专业组、6位组代号、组内1~6专业排序+服从调剂）。**为哪个省出表先按该省系统实际录入字段适配**
- `references/province-rules.md` — 找到考生所在省的**志愿填报规则**（平行志愿 vs 顺序志愿、院校专业组 vs 专业(类)+院校、可填数量、是否服从调剂、批次设置）
- `references/cross-verification-protocol.md` — **多源交叉验证协议** + 经典数据陷阱（尤其「中外合作组位次陷阱」）
- `references/value-maximization.md` — **价值最大化策略（优化层，非强制）**：同位次选更高排名/招牌、就业出口/职业确定性（好就业专业/专业排名/央国企定向/体制内上岸）、公办>民办>合作、利用信息差挖特殊招生计划。永远让位于红线

### Step 2 — 取真实官方数据（红线 1/2/4）
按 `references/data-source-registry.md` 的省级官方源 + 院校官网，采集：
1. 考生位次定位（该省**当年一分一段表**，**强制只取「普通类」对应科类那张表——普通类·首选物理 或 普通类·首选历史**；绝不能用「总分一分一段统计表（含艺术/体育/技能高考混排）」的位次，混排会让位次显著偏前、系统性误判档位）
2. 候选院校近三年（2023/2024/2025）在本省该科类**普通组**录取最低分/最低位次（区分专业组）
3. 各省当年批次线 / 本科线 / 特殊招生线（**必须核实，任务书给的数字常有误**）

> 采集纪律：每个数据点标注**来源 + 年份**；同一关键数据**至少 2 个独立来源**交叉验证；矛盾时标注争议并以官方为准。
> 取数手段降级链（强制）：`web_fetch` 直取 → 不可用/超时/被反爬拦则切**联网搜索（优先 web-search-plus skill）** → 内置 web_search → x_search；**多种方式都拿不到才写「未查到」**（详见 data-source-registry）。

### Step 3 — 位次推演定档（红线 3/5）
对每所候选院校，用 `methodology-rank-anchor.md` 的模型计算：考生位次 vs 该校近三年录取位次的差值 → 落入冲/稳/保哪一档。**剔除"超出能力范围"的伪冲档。**
> 跨年比较强制走**等位分换算法**（详 methodology 同名章）：今年分→今年位次→去年同位次的分（等位分）→用等位分比去年录取；绝不拿今年原始分直接比去年录取线。

### Step 4 — 结合偏好筛选 + 专业组细化 + 价值最大化
- 套用用户偏好（城市/专业/排除项）过滤
- 区分「热门高分组」与「压线冷门组」，提醒：**好专业的分远高于校线最低位次**，别拿最低位次判断能进好专业
- 每个专业组标注是否**中外合作/高学费**
- 按 `references/value-maximization.md` 做价值最大化（在数据真实+够得着前提下）：①同位次优先更高排名/更硬招牌 ②就业出口/职业确定性（好就业专业方向、该校该专业排名高、央国企定向/订单班、公费师范/免费医学生/优师专项/基层上岸等体制内出口） ③办学性质 公办>民办>合作 ④核实资格后挖特殊招生计划作信息差机会

### Step 5 — 出表 + 自检门控
按 `references/output-template.md` 出三档志愿表，**并必出一份按 `references/system-entry-format.md` 该省格式的「可直接录入草表」**（考生原样抄进官方系统：逆序号/院校专业组代号/组内1~6专业排序/服从调剂逐项齐备）。发布前过 `references/pitfalls-checklist.md` 全部检查点。

## 输出门控（不过不出表）
出表前必须逐条确认（任一不过 → 打回）：
- [ ] **（省份适配闸，2026-06-28）开工前已跑完省份 onboarding 协议，12 项规则卡逐项有值或明标「未查到」且带来源**；投档模式/可填志愿数/代号规则/填报系统均为**考生当年该省官方**，未套用别省/去年
- [ ] 每所院校都有近三年（≥2年）真实录取位次，且标了来源+年份
- [ ] 每个关键位次都经 ≥2 源交叉验证（或已标注"单源/未查到"）
- [ ] 当年批次线/本科线已核实（不直接采信任务书或用户口述）
- [ ] 冲/稳/保档位均由位次差推演，无"超出能力范围"伪冲档
- [ ] **（线差双锚点闸）每所院校/专业组都同时给出「位次」与「线差（录取分−当年本科批线）」两个指标**，且与考生线差交叉校验；位次与线差背离的已停下核查原因并标注，未凭单一指标定档
- [ ] **（概率闸）录取概率/定档已综合三因子**（位次差 + 该校本省招生名额增减 + 本省考生数变化），未查到的因子已如实标注且未拿猜测值拔高概率；无“XX%”伪精确概率
- [ ] 已区分普通组 vs 中外合作组位次，未被虚低位次误导
- [ ] **（真实专业组位次闸）每个专业组都用「该专业组自己的真实录取位次」定档，未用校线最低位次冒充热门专业（计算机/AI/电气/临床等）**；凡"热门专业位次≈全校最低位次"的已复核纠正；拿不到组位次的已如实标注而非顶替
- [ ] **（一分一段口径闸）考生位次取自「普通类（首选物理/历史）」专表，未误用含艺术/体育/技能高考的总分混排表**
- [ ] **（跨省闸）每所院校用的都是「该校在考生所在省」的录取数据，且该校当年在考生省有相应招生计划**（无计划即报不了，剩跨省数据一律废弃）
- [ ] （优化层）同位次已优先更高层次/招牌；已考量就业出口/职业确定性（专业就业前景/专业排名/央国企定向/体制内通道）；办学性质已标注且公办优先；符合资格的特殊计划已作信息差机会列出，且不符资格的专项低位次未计入
- [ ] 所有"未查到/单源/争议"数据已如实标注，无杜撰
- [ ] 每档都提醒"服从调剂"（平行志愿退档风险）
- [ ] 已提醒"2026 招生计划册发布后复核专业组代号"
- [ ] **（风格闸）言简意赅·篇幅克制**：表格为主、每校1~2行、精简版不超一屏；无凑字/废话/重复
- [ ] **（保守闸）冲稳保默认按保守口径**（冲靠前3%~8%、稳−3%~+5%、保靠后8%~25%；稳+保占有80%），未经用户明确授权未抬高冲档比例；位次差 >8% 的伪冲档已剔除
- [ ] **（可录入闸）已出一份符合本省志愿系统的可直接录入草表**：逐行对应系统志愿位，含逆序号+院校专业组代号+组内1~6专业排序+服从调剂；代号拿不准的已标「代号待核」而非编造；已标「代号以当年招生计划册/系统为准」免责
- [ ] **（梯度闸）草表顺序号 1→N 录取位次单调递减无倒挂**（冲在前、保在后），相邻志愿位次差合理，未扎堆同档
- [ ] **（风格闸）客观·数据支撑**：无夸张词（封神/碑压/最强/稳了等）、无主观断言（我认为/肯定能上），每个结论背后有位次/年份/学科数据支撑（详见 output-template 风格纪律）

## 交付话术
出表时附一句数据可信度声明，例如：
> 🟢 校线位次(近三年)、国家级一流专业：官方/多源可信。🟡 个别专业组精确分：部分三方汇总，趋势可靠，精确值以官方为准。🔴 未发布的当年招生计划：本表不依赖。
