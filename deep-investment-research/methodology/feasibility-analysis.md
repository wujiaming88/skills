# feasibility-analysis.md — 投资可行性分析框架

> 引用 research-core.md。用途：判断一个方向/主题/标的值不值得投，逻辑链是否成立。
> 这是"该不该参与"的把关，不是"怎么买"。

## 研究目标定义
明确要评估的对象：一个主题（如"2万亿AI基建该不该参与"）/ 一个方向（如"创新药出海"）/ 一个标的。

## 可行性评估结构

### 1. 投资逻辑链是否成立
- 把"为什么能赚钱"拆成完整链条，逐环节质询
- 哪个环节是假设、哪个是事实？最弱的环节在哪？
- 链条断在哪里 → 整个逻辑不成立

### 2. 是真机会还是已被定价（红线核心）
- 这件事市场知道吗？已经涨了多少/price in 多少？
- 区分：**利好兑现前**（机会）vs **利好已抢跑**（接盘）
- 一阶逻辑（人人能看到，已定价）vs 二阶/三阶（还没被充分认知）

### 3. 空间与弹性
- 假如逻辑成立，能赚多少？（上行空间量化）
- 对哪个环节弹性最大？（同一主题不同环节弹性差异巨大）

### 4. 确定性与时间成本
- 逻辑兑现的确定性多高？
- 需要等多久？时间成本（机会成本）是否划算
- 有没有明确的催化剂触发兑现

### 5. 风险收益比
- 下行风险 vs 上行空间，比值是否有吸引力
- 最坏情况能承受吗

## 三种结论（必须明确，红线4）
- **值得投**：逻辑成立 + 未充分定价 + 风险收益比好 + 有催化 → 说明最优参与环节、关键假设、跟踪信号
- **不值得投**：给出具体一句话原因（逻辑断在哪/已抢跑/弹性不足/风险不可控/时间成本太高）
- **有条件值得**：当前不行，但若出现 X 条件则成立 → 列出触发条件，进观察清单

## 红线
- 写不出"为什么不是已被定价" → 倾向不值得投
- 不因"题材大/故事好"就给值得投——故事≠预期差
- 紧扣"值不值得投"这个问题，不发散成泛行业综述

## 质量对照（肤浅 vs 深刻）
**❌ 肤浅**："这是万亿级大赛道，空间巨大，值得投。" —— 题材当结论，没回答"是否已被定价""哪个环节"。
**✅ 深刻**："逻辑链：A政策→B需求→C标的，最弱环节是B（需求兑现依赖补贴落地，存疑）；一阶受益股（服务器）1个月+102%已抢跑，二阶环节（电力/IDC）估值仍低未price in；最优risk:reward在二阶；确定性中、需等细则、时间窗口1-3月。结论：有条件值得，触发条件=招标落地。" —— 拆链条、查定价、定环节、给条件。
