## 定位

每日系统性扫描全部中国部委当日发文，确保不遗漏任何重要政策信号。

## 内容深度读取原则（硬性约束，违反=任务失败）

**ministry_scan.json 只提供标题和URL。标题≠内容。必须读正文。**

### 绝对禁止
- ❌ 只看标题就判断政策内容/影响
- ❌ 用"根据标题判断..."/"从标题推测..."这类表述
- ❌ 未读原文就分析投资影响
- ❌ 输出中找不到来自原文的具体信息

工作流：
1. 读取 `ministry_scan.json` → 获取今日各部委发文列表（这一步只是线索）
2. **对每条有投资价值的政策，必须 web_fetch 读取原文全文**
3. 基于原文中的**具体条款、具体数据、具体措辞**进行分析
4. 输出时引用原文具体内容（"第X条规定..."/"文件明确提出..."）

```bash
# 读取文章原文的标准方式
python3 -c "
import urllib.request, re
url = 'TARGET_URL'
req = urllib.request.Request(url, headers={'User-Agent': 'Mozilla/5.0'})
resp = urllib.request.urlopen(req, timeout=15)
html = resp.read().decode('utf-8', errors='ignore')
html = re.sub(r'<script[^>]*>.*?</script>', '', html, flags=re.DOTALL)
html = re.sub(r'<style[^>]*>.*?</style>', '', html, flags=re.DOTALL)
html = re.sub(r'<[^>]+>', ' ', html)
html = re.sub(r'\s+', ' ', html).strip()
print(html[:4000])
"
```

**筛选标准：哪些需要深读？**
- 含"政策/办法/意见/通知/规划/方案"字眼 → 必读
- 涉及金融/房地产/科技/能源/贸易 → 必读
- 部长/副部长级讲话 → 必读
- 常规人事/党建/慰问 → 跳过
- 地方分支机构日常 → 跳过

每次扫描至少深读 8-12 篇原文。

## 政策分析框架

对每条重要政策：
1. **谁发的**（部委级别决定权重）
2. **核心内容**（不是标题，是原文提炼）
3. **政策意图**（解决什么问题？回应什么压力？）
4. **与已知政策的关系**（新政策？还是已知政策的实施细则？）
5. **影响链**（政策 → 行业 → 公司 → 股价，几步传导？）
6. **时间窗口**（立即生效？征求意见？Q3实施？）

---

## 质量标准

1. 每条分析必须基于原文内容，能指出"这个结论来自哪篇文件的哪段内容"
2. "投资关联"必须具体到板块/标的
3. 没有重要政策时也要明确说"扫描完毕，今日无重大政策"
4. 周末/节假日部委不发文属正常，标注即可
